# DisTrainer Setup Guide

## Directory Structure

```
DisTrainer/
├── models/
│   ├── arcee-trinity/          # Base Trinity-Mini model (26B MoE, shared)
│   │   ├── config.json
│   │   ├── model.safetensors.*
│   │   ├── tokenizer_config.json
│   │   └── ...
│   └── parallelization/        # FSDP2 code modules
│       ├── dense_model.py
│       ├── moe_model.py
│       └── freezing.py
│
├── policies/                    # LoRA adapters (generated by trainer)
│   ├── policy-0-initial/       # Initial adapter from Modal
│   ├── policy-1-20260211_120000/
│   ├── policy-2-20260211_130000/
│   └── latest_adapter -> policy-2-...  # Symlink (auto-created)
│
├── config/
│   └── train_config.toml       # Training configuration
│
└── scripts/
    └── download_assets.py      # Download base model + initial adapter
```

## Quick Start

### Step 1: Download Assets

Download the base model from HuggingFace and initial adapter from Modal:

```bash
cd serving/DisTrainer

# Install dependencies
pip install huggingface_hub modal transformers

# Authenticate
huggingface-cli login  # Optional for public models
modal setup            # Required for Modal access

# Download both model and adapter
python scripts/download_assets.py

# Or download selectively
python scripts/download_assets.py --skip-model     # Only adapter
python scripts/download_assets.py --skip-adapter   # Only model
```

This downloads:
- **Base model**: `arcee-ai/Trinity-Mini` from HuggingFace → `models/arcee-trinity/`
- **Initial adapter**: `models/trinity-triton-sft-vllm/final_model` from Modal → `policies/policy-0-initial/`

### Step 2: Verify Structure

```bash
ls -la models/arcee-trinity/      # Should have config.json, model files, tokenizer
ls -la policies/policy-0-initial/ # Should have adapter_config.json, adapter_model.safetensors
```

### Step 3: Configure Training

Edit `config/train_config.toml` if needed:

```toml
[model]
name = "./models/arcee-trinity"  # Local path to base model
adapter_path = "./policies/policy-0-initial"  # Initial adapter

[checkpoint]
save_dir = "./policies"  # New policies saved here
```

### Step 4: Start Training

```bash
# 4 GPU setup (2 trainer GPUs)
export CUDA_VISIBLE_DEVICES=2,3
torchrun --nproc_per_node=2 -m DisTrainer.train --config config/train_config.toml

# 8 GPU setup (4 trainer GPUs)
export CUDA_VISIBLE_DEVICES=4,5,6,7
torchrun --nproc_per_node=4 -m DisTrainer.train --config config/train_config.toml
```

## How It Works

### Base Model (Shared)

- **Location**: `models/arcee-trinity/`
- **Source**: HuggingFace (`arcee-ai/Trinity-Mini`)
- **Used by**: Both DisGenerator and DisTrainer
- **Downloaded**: Once (26B model, ~50GB)

### Policies (LoRA Adapters)

- **Location**: `policies/`
- **Initial**: `policy-0-initial/` from Modal (vLLM-compatible, attention-only)
- **Generated**: `policy-1-*`, `policy-2-*`, etc. by DisTrainer
- **Symlink**: `latest_adapter` auto-updated to newest policy
- **Used by**: DisGenerator for inference (hot-swappable)

### Training Flow

```
1. DisTrainer loads:
   - Base model: models/arcee-trinity/
   - Initial adapter: policies/policy-0-initial/

2. Training runs on batch data

3. DisTrainer saves checkpoint:
   - New policy: policies/policy-1-20260211_120000/
   - Updates symlink: latest_adapter -> policy-1-20260211_120000
   - Creates signal: policies/.policy_ready

4. DisGenerator detects:
   - PolicyManager watches for .policy_ready
   - Hot-swaps to new policy (zero downtime)
   - New trajectories use updated policy

5. Repeat: Training continues with next batch
```

## Troubleshooting

### Model not found

```bash
# Check if model exists
ls -la models/arcee-trinity/config.json

# If missing, download
python scripts/download_assets.py --skip-adapter
```

### Adapter not found

```bash
# Check if initial policy exists
ls -la policies/policy-0-initial/adapter_config.json

# If missing, download from Modal
python scripts/download_assets.py --skip-model
```

### DisGenerator can't find policies

The generator looks in `../DisTrainer/policies/` relative to its location. Ensure:

```bash
# From AsyncRL/serving/
ls -la DisTrainer/policies/policy-0-initial/  # Should exist
ls -la DisTrainer/policies/latest_adapter/    # Symlink (created after first checkpoint)
```

### Manual symlink creation

If needed:

```bash
cd DisTrainer/policies
ln -sfn policy-0-initial latest_adapter
```

## Configuration Reference

### DisTrainer (`config/train_config.toml`)

```toml
[model]
name = "./models/arcee-trinity"  # Base model path
adapter_path = "./policies/policy-0-initial"  # Initial adapter

[checkpoint]
save_dir = "./policies"  # Where new policies are saved
```

### DisGenerator (`DisGenerator/config.py`)

```python
DISTRAINER_POLICIES_DIR = os.getenv(
    "DISTRAINER_POLICIES_DIR",
    "../DisTrainer/policies"  # Where to find policies
)
```

## Notes

- Base model is ~50GB (26B parameters)
- Initial adapter is ~100MB (LoRA weights only)
- New policies are ~100MB each
- `keep_latest_k = 3` keeps only 3 most recent policies
- Symlink enables hot-swap without path changes
